Why you aren't showing up yet

Your code is not the bottleneck. The audit below will confirm the on-page work is in good shape — structured data, metadata, llms.txt and AI-crawler permissions are all already in place.

The actual blocker is that annaprudential.com is a new domain with no inbound links. Google will not rank a site it has barely crawled, and ChatGPT, Gemini and Perplexity can only cite pages that are already indexed and corroborated somewhere else on the web. That is a distribution problem, and no amount of meta-tag tuning solves it.

So use this dashboard in two halves: Auto-optimise keeps the code perfect on its own, and the Free traffic plan tab is the part that actually moves the needle. Start there.

Auto-optimise

This rewrites your HTML to fix everything that can be fixed deterministically: missing canonicals, social meta, viewport, lazy-loading, breadcrumb schema, FAQ schema built from Q&A already on the page, plus a full regeneration of robots.txt and sitemap.xml.

What it will never do: invent content. No fabricated descriptions of things not on the page, no made-up statistics, no fake reviews or dates. On a licensed insurance site that is a compliance risk, and search engines penalise it anyway. Findings that need real writing stay in the report for you.

Always preview first. Everything is under git, so git diff shows exactly what changed and git checkout . undoes all of it.
